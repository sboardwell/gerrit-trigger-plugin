When enabled, webhook requests must include a valid HMAC-SHA256 signature to be accepted.

The signature must be provided in the X-Gerrit-Signature HTTP header in the format: sha256=<hex-signature>

If this option is disabled (unchecked), the HMAC secret configuration is ignored and requests are accepted without signature validation.

Recommended: Enable this option if you have configured an HMAC secret to ensure payload integrity and authenticity.

Security Benefit: HMAC signature validation is more secure than simple token authentication because it:

  • Validates the entire payload content (detects tampering)
  • Prevents replay attacks when combined with timestamps
  • Provides cryptographic proof of authenticity

Use case for disabling: You may want to disable this if you're using only token-based authentication or IP whitelisting, without HMAC signature validation.
