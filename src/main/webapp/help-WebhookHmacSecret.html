A shared secret key used to validate the integrity of webhook payloads using HMAC-SHA256 signatures.

When configured, Gerrit will compute an HMAC signature of the webhook payload using this secret and include it in the X-Gerrit-Signature HTTP header as: sha256=<signature>

Jenkins will verify the signature to ensure:

  • The payload was sent by an authorized source (has the secret)
  • The payload was not tampered with during transmission

Configuration in Gerrit: Configure the same HMAC secret in your Gerrit server's webhook settings to enable signature validation.

Note: HMAC signature validation provides stronger security than simple token authentication as it validates payload integrity.

Security: The secret is stored encrypted in Jenkins. Use a strong, random value (at least 32 characters recommended).
